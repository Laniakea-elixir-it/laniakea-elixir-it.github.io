Dear Users,

the package CentOS ca-certificates package needs to be manually updated in order to restore data upload from HTTPS sources.

Currently, the upload module will display the following error if data are uploaded through HTTPS link: Unable to fetch https://data-url
urlopen error [SSL: CERTIFICATE_VERIFY_FAILED] certificate verify failed (_ssl.c:618)

To update the package login on your instance through SSH and type:
sudo yum update -y ca-certificates
After the package update Galaxy will import dataset correctly.

New Galaxy instanes will be automatically updated.

The Laniakea team.
